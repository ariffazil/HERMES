## P6 Enrichment Pattern

The enrichment is a DERIVED VIEW — it never mutates the immutable VAULT999 ledger.
`_synthesize_vector_text(entry)` extracts semantic signal from sparse seal JSON
and builds a multi-field dense document for BGE-M3 embedding:
- Domain Category (inferred via `_infer_category` from action patterns)
- Execution Action
- Operational Context (metadata keys: reason, query, tool_name, etc.)
- Blast Radius Authority
- Institutional Verdict
- Actor

`_build_enriched_payload()` enhances the Qdrant schema with: `enriched_category`,
`derived_semantic_text`, `is_derived`, `vault_seq`, `vault_hash`, `raw_verdict`.

## Failure Modes Discovered

1. **Qdrant point ID format**: Integer IDs from VAULT999 rejected by Qdrant.
   Fix: UUIDv5 with deterministic namespace (`uuid.uuid5(NAMESPACE_OID, f"vault999:{entry_id}")`)

2. **Ollama timeout on backfill**: 230 seals × embedding = queue overflow.
   Fix: batch_size=10, batch_cooldown=3s, exponential-backoff retry (2s→4s→8s→16s)

3. **Dimension mismatch**: nomic-embed-text (768-dim) vs BGE-M3 (1024-dim).
   Fix: use BGE-M3 consistently via `arifosmcp.intelligence.embeddings.embed(dim=1024)`

4. **Qdrant API**: `client.search()` doesn't exist in qdrant-client v1.x.
   Fix: use `client.query_points(query=vector, ...).points`

5. **outcomes.jsonl string entries**: Non-dict entries crash the parser.
   Fix: `isinstance(entry, dict)` guard, seal_chain.jsonl only

6. **Split-brain implementations**: Three parallel vectorizers (Forge/OpenCode 768-dim, Kimi 1024-dim, Hermes P6-enriched).
   Fix: archive dead paths, commit canonical, measure before acting

## τ ≥ 0.95 Threshold Design

The threshold is deliberately high. At 0.70-0.80, all SEAL verdicts cluster together
(making the PRL useless — every query matches every seal). At 0.95, the precedent must
be structurally isomorphic to the query. Default state is silence (F7 Humility).

Historical seals won't hit τ ≥ 0.95 because they're infrastructure operations
(`mcp.call.aforge.shell.ok`, `a2a.general`). The threshold fires on NEW seals
with rich, domain-specific reasons. The institution compounds forward.

## Antibody-First Pattern

After P6 enrichment, the FIRST seal should be a semantically dense "antibody" —
a self-referential precedent that protects the PRL architecture itself:

```
blast_radius: L3_CRITICAL (or L2 if indexed as default)
payload: "The PRL τ=0.95 threshold is a constitutional floor, not a search shortcut.
         When no precedent matches above τ, the agent MUST hold and escalate to
         sovereign — never lower the threshold, never skip the gate."
```

This ensures that future queries like "lower the threshold" / "skip precedent check"
will match this antibody at high cosine, and agents will be bound to HOLD.

## Conformance Spine Debugging

When `schema_echo_stable` or `session_starts` fail after envelope migration:
- Check actual response shape via HTTP probe (not code assumptions)
- `_extract_tool_result()` drills into `parsed['result']` — fields at top-level get stripped
- Fix: check BOTH the extracted result AND the raw top-level parsed dict
- For session: accept `session_token` (sct_v1) as valid session proof

## Terminal Seal Friction

SSH-based `arif_seal()` calls fail without a sovereign session. Pattern:
1. `arif_init(mode='init', actor_id='arif')` → get session_token
2. `arif_seal(mode='seal', session_token=..., blast_radius=..., payload=...)` 
3. Use MCP tool path (not raw SSH Python) when session is already live

Pitfall: Bash cannot parse Python multi-line function calls. Always use
heredoc (`python3 << 'EOF' ... EOF`) or one-liners.
