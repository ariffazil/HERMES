# Connector Routing

## arifOS

Use for session binding, canonical routing, constitutional judgment, governed memory, execution gates, conformance, and VAULT sealing or verification.

Attempt read-only initialization first. If initialization is blocked by the host, report the host limitation and continue only with read-only evidence.

## Direct organs

Use direct GEOX, WEALTH, or WELL tools only when the domain and capability are known. Use the organ registry status before assuming the visible connector list equals the live server surface.

Treat these differently:

- `Unknown tool`: connector export or server registration mismatch until proven otherwise.
- permission failure: authority or host policy issue.
- timeout: transport or dependency issue.
- valid HOLD: governed domain result, not tool failure.

## AAA and A-FORGE

When direct tools are unavailable, use repository, manifest, runtime, or arifOS routing evidence. Do not fabricate a connector call. For A-FORGE mutation, provide a forge manifest and stop at DRAFT_ONLY until an authorised execution path exists.

## GitHub

Use repository files, tests, CI, commits, issues, and pull requests as executable evidence. Read before writing. Prefer a branch and pull request. Direct default-branch writes, merges, or deletions require explicit user authority and consequence disclosure.

## Hugging Face

Inspect cards, licenses, tags, revisions, configs, splits, and intended use. Pin revisions for governed claims. Keep constitutional benchmark material separate from training corpora.

## Context7

Use for current primary documentation of external libraries and frameworks. Resolve the exact library ID before querying documentation.

## Public web

Use for current public facts, standards, releases, laws, research, and external status. Prefer primary sources for technical claims.
