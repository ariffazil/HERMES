# FI Three-Way Contrast — Session Trace (2026-07-18)

## Summary

Full three-way comparison of OpenCode (FI-001), Claude Code (FI-002), Kimi Code (FI-008) across 13 registries/surfaces: forge_instruments.yaml, ROOT_AGENT_CONFIG.yaml, AAA_AGENTS_REGISTRY.json, agents.yaml, agent-card.json copies (×3+ per instrument), MCP surfaces, model setups, subagent policies, skills indexes, hooks/services/telegram, toolbench registries, binary/config paths, and unique features.

## Key Patterns Discovered

### 1. FI Number Directory Mismatch (Kimi Code)
- **Finding:** Directory is `fi-003-kimi-code` but agent card internally says `"id": "FI-008"`
- **Root cause:** CIV-33 originally assigned FI-003; later reorganized to FI-008. Directory wasn't renamed.
- **Symptom:** `skills.json` says `"fi": "FI-003"` but forge_instruments.yaml and all agent cards say FI-008
- **Known:** Documented in `kimi-code-model-management` skill as known bug

### 2. Agent Card Version/Model Drift
- **OpenCode (FI-001):** 3 cards found — versions 1.15.0, 3.1.0, 3.1.0; primary model varies between deepseek-v4-pro and mimo-v2.5-pro
- **Claude Code (FI-002):** 2 cards found — versions 2.1.160, 2.0.0; one says deepseek-v4-pro, the other says claude-sonnet/opus-4
- **Kimi Code (FI-008):** 4 cards found — versions 1.47.0, 2.0.0, 2.1.0, 1.47.0

### 3. Registry Presence Asymmetry
| Registry | OpenCode | Claude Code | Kimi Code |
|----------|----------|-------------|-----------|
| forge_instruments.yaml | ✅ Full | ✅ Full | ✅ Full |

Claude Code is the ONLY forge instrument with a full A2A registry entry. OpenCode has only an indirect binding via `777-forge`.

### 4. Contrast Skills Only Exist for One Instrument
OpenCode has 12 contrast skills (4 roles × 3 tiers: AGI/ASI/APEX). Claude Code and Kimi Code have NONE declared in forge_instruments.yaml. Kimi Code has 7 in its local skills directory but they aren't registered in the FI entry.

### 5. Systemd/Telegram Only for One Instrument
Only OpenCode has systemd services (`opencode.service`, `opencode-bot.service`) and a Telegram bot (`@777_FORGE_bot`). Neither Claude Code nor Kimi Code have either.

### 6. Claude Code Has the Richest Hook Infrastructure
9 hook types with dedicated scripts: SessionStart, PreToolUse, PostToolUse, PostToolUseFailure, UserPromptSubmit, PermissionRequest, PreCompact, PostCompact, Stop. OpenCode and Kimi Code have hook dirs but no documented scripts.

### 7. A-FORGE Transport Divergence
- OpenCode: HTTP (:7072) — ✅
- Claude Code: HTTP (:7072) — ✅
- Kimi Code: STDIO (`/root/.arifos/agents/kimi/mcp-launchers/aforge.sh`) — deliberately chosen for reliability with single-session SDK

### 8. CIV-33 Directories Non-Existent
No CIV-33 subdirectories exist under `/root/AAA/agent-cards/`. CIV-33 content lives under `/root/AAA/domain-atlas/` instead (math profiles). CIV-333 directory nesting scheme (identity/functions/extensions/harnesses/organs/pillars) exists at agent-cards level but Kimi's harness is under wrong FI directory.
