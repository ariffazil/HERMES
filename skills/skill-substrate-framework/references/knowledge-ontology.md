# Knowledge Ontology — 8 Domains × 3 Axes

Every domain has: INVARIANT (what never changes), BRIDGE (how it connects), CONTRAST (internal tension).

## The 8 Domains

### Φ PHYSICS
- **Invariant:** SUSTAIN + VERIFY — Conservation laws don't negotiate.
- **Bridges:** → Math (equations), → Computation (simulation), → Biology (biophysics), → Cognition (entropy of thought)
- **Contrasts:** Determinism vs Emergence | Reductionism vs Holism | Precision vs Applicability
- **Agent skills:** geo-grounding, geo-evidence, geo-petrophysics, eng-docker-entropy

### Μ MATHEMATICS
- **Invariant:** REDUCE + VERIFY — Proof is the only currency.
- **Bridges:** → Physics (language of physics), → Computation (algorithms), → Linguistics (formal grammar), → Cognition (Bayesian reasoning)
- **Contrasts:** Completeness vs Consistency (Gödel) | Abstraction vs Utility | Discrete vs Continuous
- **Agent skills:** ker-eureka, met-plan-dag, met-critique

### Λ LINGUISTICS
- **Invariant:** GUARD + SHADOW — Language shapes thought.
- **Bridges:** → Math (formal semantics), → Computation (NLP/LLMs), → Cognition (Sapir-Whorf), → Social (power dynamics)
- **Contrasts:** Structure vs Meaning | Prescription vs Description | Universal vs Particular
- **Agent skills:** gov-nusantara, con-bias, con-trust, met-summarize

### Β BIOLOGY
- **Invariant:** SUSTAIN + REVERSE — Life is thermodynamic persistence.
- **Bridges:** → Physics (biophysics), → Math (population dynamics), → Cognition (neuroscience), → Computation (genetic algorithms)
- **Contrasts:** Genotype vs Phenotype | Selection vs Drift | Individual vs Population
- **Agent skills:** geo-readiness, mem-dream, eng-docker-thermo

### Ψ COGNITION
- **Invariant:** SHADOW + REDUCE — Self-deception is the default.
- **Bridges:** → Math (Bayesian brain), → Linguistics (inner speech), → Biology (neuroscience), → Computation (AI/ML)
- **Contrasts:** Rationality vs Heuristics | Consciousness vs Computation | Individual vs Collective
- **Agent skills:** con-shadow, con-boundary, con-membrane, met-rsi

### Κ COMPUTATION
- **Invariant:** VERIFY + REDUCE — A program is a proof.
- **Bridges:** → Math (Turing machines), → Physics (quantum computing), → Linguistics (programming languages), → Cognition (AI)
- **Contrasts:** Correctness vs Performance | Abstraction vs Control | Centralized vs Distributed
- **Agent skills:** eng-precommit, eng-mcp-federation, met-evals, met-verify

### Σ SOCIAL
- **Invariant:** GUARD + SHADOW — Institutions are frozen power.
- **Bridges:** → Linguistics (discourse analysis), → Biology (evolutionary psychology), → Math (game theory), → Cognition (groupthink)
- **Contrasts:** Order vs Freedom | Meritocracy vs Structural Advantage | Transparency vs Privacy
- **Agent skills:** gov-governance, gov-floors, gov-authority, gov-nusantara

### Α ART
- **Invariant:** GUARD + SUSTAIN — Beauty is meaning before language.
- **Bridges:** → Math (fractals, harmony), → Linguistics (poetry, metaphor), → Cognition (aesthetic experience), → Social (art as resistance)
- **Contrasts:** Craft vs Expression | Original vs Universal | Autonomous vs Commissioned
- **Agent skills:** con-least-power, con-zen-memory, met-wiki

## How to Use

1. When creating a domain skill, declare which knowledge domain(s) it bridges to
2. When auditing coverage, map skills against domain keywords
3. Gaps <25% in any surface need dedicated skills
4. Bridge skills (spanning 3+ domains) are the strongest assets — protect during pruning
5. Every domain has an internal contrast — don't resolve it, hold it productively
