# Geoscience Artifact Rigor — Standing Instruction

> Origin: PETRONAS exploration geologist review — "tak cukup geology."
> The artifact had epistemic tags but lacked actual technical substance.

## Hard Rules (non-negotiable)

### 1. Tags are not content
CLAIM/HYPOTHESIS/SCHEMATIC = wrapper around a claim, not the claim itself. Every tagged item must carry the actual data point, number, reference, or mechanism. Never let a colored tag stand in for evidence.

### 2. No cartoon geometry for technical audiences
Do not render generic sine-wave "basin" shapes, blob "anticlines," or arrow-only "overpressure" icons. If real seismic character, well logs, structure maps, or published figures exist — reference or reproduce them with citation. If no real data exists, say so explicitly. Do not fill gaps with authoritative-looking schematics.

### 3. Quantitative claims need the curve, not just the conclusion
- Timing claims (e.g., "oil expelled early, gas late") → must cite actual maturity indicator (Ro, Tmax, burial history) or flag as UNSUPPORTED ASSUMPTION
- Reservoir/seal/trap claims → need closure area, net-to-gross, or volumetric range — not just "reservoir here" cartoon
- Age claims → cite the calibration source (biostrat, radiometric, GPTS)

### 4. Terminology precision check
Cross-check every named geological entity, terrane, or structural term against established usage in cited literature. Flag conflation of distinct concepts (e.g., crustal block name vs. process acting on it). State the check was done.

### 5. Name the upgrade path for every HYPOTHESIS
For every HYPOTHESIS/PLAUSIBLE item, name the specific dataset, well, or measurement that would move it to CLAIM. The reader should know exactly what's missing.

### 6. Age/stage precision must match data resolution
Do not present a stage duration more narrowly than the underlying dataset resolves. If an age is an onset bound for a longer process, say so. Don't let stage boxes imply false precision.

### 7. Self-check before delivery
Ask: "Would a subsurface geologist with access to real well/seismic data accept this as technical content, or would they say this is dressing without depth?" If the honest answer is the latter:
- Add real data with source citation
- Or explicitly scope as "conceptual framing only" — labeled up front

### 8. Framework ≠ Finding
The epistemic-tagging system (CLAIM/PLAUSIBLE/HYPOTHESIS) is a governance layer for tracking confidence over time. It governs geological content; it does NOT generate geological content. Never let tagging elegance create false impression of rigor.

## Checklist Before Shipping Any Geoscience Artifact

- [ ] Every CLAIM has a number, reference, or mechanism behind it
- [ ] No cartoon geometry where real data should be
- [ ] Quantitative claims cite the actual curve/measurement
- [ ] Terminology cross-checked against cited literature
- [ ] Every HYPOTHESIS names the upgrade dataset
- [ ] Age precision matches data resolution
- [ ] Self-check passed: a working geologist would accept this
- [ ] Epistemic tags sit ON TOP of real substance, not instead of it

## Red Flags (stop and fix)

- Schematic cross-section with no real geometry or citation
- "Overpressure" shown as an arrow without burial history or Ro data
- "Reservoir" shown as a colored blob without NTG, porosity, or extent
- Age assigned without citing calibration source
- Named structural trend without citing the defining paper's figure number
- "Claim" tag on something that's actually an assumption
