## Session Flow

1. **User provides URL** → Malay Mail article on Khairul Akmal espionage case
2. **Phase 1-2:** Fetch article + search related coverage → build chronological timeline (15 dates, ≥2 sources each)
3. **Phase 3:** Detection path → WBC → Cyber Security → HR → IR → police report
4. **Phase 4:** Institutional relationship pattern → who sits behind each trigger?
5. **User provides Shell MDS context** → "bank guarantee trigger" → build second case file

3. **BOD resigned during crisis peak** — Ibrahim Baki (Sarawak director) resigned same month as BG call
4. **Profit declined 3 years straight** — -21%, -32%, -18% under Tengku Taufik
5. **Simulative exploitation pattern** — external actor exploiting institutional weakness under neutral guise

## Lesson: Convergent vs Sequential Thinking

The agent processed all data sequentially but didn't see the pattern until the user pointed it out:
- Agent: "Here's the timeline, here's the MBR data, here's the BOD data" (sequential)
- User: "Shell never sued in 60 years. BOD thin. Profit jatuh. Ibrahim Baki resign. Masa tu lah Shell strike." (convergent)

**Takeaway:** After every batch of data, ask "what converges here?" Don't just build timelines — look for vulnerability windows where multiple stress signals fire simultaneously.
